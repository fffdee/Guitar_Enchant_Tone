## 1. 项目简介与目标

ESP-GT-XFORM 旨在用**一个条件化神经网络**把电吉他演奏实时转换为 BASS / 钢琴 / 提琴等多种乐器音色，
切换乐器仅需更换一个 8 维嵌入向量，零模型重载。核心技术路线为 **DDSP（可微分数字信号处理）**：

- 网络只预测**缓慢变化的合成参数**（30 维谐波幅度 + 10 维噪声子带），
- 最终音频由**数学完美的合成器**生成，天然规避神经声码器的相位伪影，计算量极低。

PC 阶段的目标是**先在电脑上闭环验证**：数据生成 → 特征提取 → 标签分析 → 条件网络训练 →
DDSP 合成 → 多乐器切换 → 导出（embedding / 归一化 / checkpoint / C 头文件）。

## 2. 关键设计决策与技术栈

| 决策 | 内容 | 原因 |
|---|---|---|
| **核心仅依赖 `numpy` + `torch`** | 不强依赖 librosa / soundfile / scipy / torchaudio | 缺包不阻塞；算法纯手写便于 1:1 移植到 ESP32 C |
| **WAV 用标准库 `wave`** | 自实现 8/16/24/32-bit、多声道读取，写出统一 16-bit | 规格固定 48k/16bit/mono，无需第三方库 |
| **特征自研** | MFCC / YIN-F0 / 频谱特征全部用 numpy 实现 | 算法可控，与 C 端数值严格一致 |
| **噪声用 10 子带 RMS** | 取代原方案的时变 LPC | 更稳定、易合成，且与分析器**精确互逆** |
| **网络输出线性（无 softplus）** | 合成时再 clamp 到非负 | 标签含大量精确 0，softplus 的 0.69 下限有害 |
| **STFT 损失用「标签域代理」** | 比较 `合成(预测参数)` 与 `合成(目标标签)` | 无需缓存原始波形即可获得音频域梯度 |
| **配置 dataclass + 可选 YAML** | 默认值写在代码内，YAML 可覆盖 | 无 pyyaml 也能完整运行 |

**运行环境**：Python 3.10（conda 环境 `tcn_cuda`），PyTorch 2.13 + CUDA 13.2，
GPU 为 NVIDIA RTX 5070 Ti。仅 `numpy` + `torch` 为必需；`pyyaml`/`tqdm`/`matplotlib` 为可选增强。

## 4. 端到端数据流

```
                       ┌─────────────────────────────────────────────┐
  原始/渲染音频          │  预处理(可选): 单声道→48k重采样→对齐→去静音→切段  │
  (任意采样率/声道)  ──▶ │  esp_xform.preprocess                         │
                       └─────────────────────────────────────────────┘
                                          │  48k/mono/对齐的 guitar & 目标 WAV
                                          ▼
   prepare_manifest.py  ──▶  manifest.csv (sample_id, source_wav, target_wav, target_instrument)
                                          │
              ┌───────────────────────────┴───────────────────────────┐
              ▼                                                         ▼
  extract_features.py                                        extract_labels.py
  源吉他 → GuitarFeatureExtractor                            目标 → DDSPAnalyzer(用源 f0 定位谐波)
  → features/<sample_id>.npz                                 → labels/<sample_id>__<target>.npz
  (features:20, f0_hz, voiced, loudness)                     (features:20, labels:40, f0_hz, instrument_id)
                                          │
                                          ▼
                                      train.py
            按 sample_id 划分 → 特征标准化 → DDSPFrameDataset(序列窗)
            → ConditionalDDSPNet(FiLM 条件) → param MAE(+可选STFT) → AdamW
            → 周期渲染 demo(各乐器+切换) → 导出(checkpoint/embedding.h/归一化.h)
                                          │
                                          ▼
                       render_demo.py / infer.py：吉他特征 → 预测 40 维参数
                       → DDSP 合成(谐波振荡器 + 子带噪声) → 48k WAV
```

**关键不变式**：源吉他与目标乐器来自同一段 MIDI（音高/时序一致），因此目标谐波用
**源吉他的逐帧 f0** 定位；训练即学习「吉他特征 → 目标乐器 DDSP 参数」的映射。

## 5. 核心库模块详解（`esp_xform`）

### 5.1 `config.py` — 配置体系 ✅
- 6 个 dataclass：`AudioConfig` / `FeatureConfig` / `DDSPConfig` / `ModelConfig` / `TrainConfig` / `PathsConfig`，聚合为 `Config`。
- 派生属性：`input_dim = feature_dim + embedding_dim`（28）、`output_dim = n_harmonics + n_noise_bands`（40）。
- `load_config(path)`：默认值 + YAML/JSON 覆盖；无 `pyyaml` 时回退默认并告警。
- `load_instruments(path)`：返回 `{instruments: {...}, target_instruments: [...]}`。
- `Config.validate()`：校验维度自洽（如 `output_dim` 一致、`film_layers` 不越界）。

### 5.2 `audio/` — 音频 I/O 与基础 DSP ✅
| 文件 | 功能 |
|---|---|
| `io.py` | `load_wav`（兼容 8/16/24/32-bit、多声道→mono、可校验采样率）、`save_wav`（16-bit PCM，自动裁剪防爆音） |
| `framing.py` | `frame_signal`（非中心、与流式帧布局一致）、`overlap_add`（加权平均 OLA，重建误差 ~9e-8）、`hann_window`、`num_frames` |
| `normalize.py` | `amp_to_db`/`db_to_amp`、`peak_normalize`、`rms_normalize` |
| `resample.py` | `resample_to`（FFT 法，主频精确保持、上下采样往返误差≈0） |

### 5.3 `features/` — 吉他 20 维特征 ✅
- `f0_yin.py`：YIN 基频（差分函数 → CMND → 绝对阈值 → 抛物线插值）。合成纯音实测 **220.00 Hz**。
- `mfcc.py`：HTK Mel 滤波器组、DCT-II、功率谱、`mfcc_frame`。
- `guitar_features.py`：`GuitarFeatureExtractor` 输出 `(T, 20)` 原始特征 + 合成对齐量（f0/响度/voiced）。

**20 维特征布局**：

| 索引 | 名称 | 说明 |
|---|---|---|
| 0 | log_f0 | 对数基频（静音以 f0_min 下限） |
| 1 | loudness_db | RMS→dB |
| 2–14 | mfcc[0:13] | 13 维 MFCC |
| 15 | spectral_centroid | 频谱质心（归一化到 [0,1]） |
| 16 | spectral_rolloff | 85% 能量滚降点（归一化） |
| 17 | voiced_flag | 有声标志 0/1 |
| 18 | zcr | 过零率 |
| 19 | spectral_flux | 相邻帧幅度谱正向差 |

> 原始特征不做标准化；训练前由 `compute_feature_stats` 统计全局 mean/std，
> 训练/推理/未来 C 端必须使用**同一套** mean/std（已随 checkpoint 与 `.h` 导出）。

### 5.4 `ddsp/` — 分析 / 合成 / 损失 ✅
DDSP 是本工程的核心，分析器与合成器构成**严格配对的编码器/解码器**。

- `bands.py`：`make_band_edges` 按 Mel 间隔划分 10 个噪声子带（分析与合成共用，保证一致）。
- `analysis.py`：`DDSPAnalyzer.analyze(target_audio, f0_hz)` → `labels (T,40)`。
  - 谐波幅度 `A_k = 2·|X_peak| / Σw`（主瓣峰值换算时域正弦幅度）。
  - 噪声子带 = 去除谐波 bin 后，残差按子带的时域 RMS（Parseval + 窗能量补偿）。
- `synthesizer.py`：
  - numpy 版（离线渲染/对应 C 实现）：`harmonic_synth_np`（相位累加加性振荡器）、
    `filtered_noise_np`（对单位 RMS 带通噪声乘逐帧增益，无接缝、RMS 可控）、`ddsp_synth_np`。
  - torch 版（可微，用于 STFT 损失）：`harmonic_synth_torch` / `filtered_noise_torch` / `ddsp_synth_torch`。
- `losses.py`：`param_loss = MAE(谐波) + w·MAE(噪声)`；`multi_scale_stft_loss`（多尺度幅度+对数幅度 L1）。

**40 维标签布局**：`[0:30]` 谐波幅度，`[30:40]` 噪声子带 RMS。

**分析↔合成一致性**（已验证）：对 0.5/0.25/0.1 三谐波信号，恢复幅度 ≈ 0.473/0.245/0.099，合成 RMS 与目标吻合。

### 5.5 `models/` — 条件 DDSP 网络 ✅
- `film.py`：`FiLM` 由嵌入生成逐通道 γ、β，`x' = (1+γ)·x + β`，**零初始化**（初期恒等，稳定）。
- `conditional_ddsp.py`：`ConditionalDDSPNet`

| 层 | 配置 | 说明 |
|---|---|---|
| 输入 | 20 特征 ⊕ 8 嵌入 = 28 | 嵌入广播到每帧并拼接 |
| Conv1D | 28→32, k3, d1 + ReLU | |
| Conv1D | 32→64, k3, d2 + **FiLM** + ReLU | |
| Conv1D | 64→128, k3, d4 + **FiLM** + ReLU | |
| Conv1D | 128→64, k3, d1 + ReLU | |
| Head | 64→40, k1（线性） | 合成时 clamp 非负 |

- 嵌入矩阵 `nn.Embedding(num_instruments, 8)` 随网络训练；切换乐器只换嵌入。
- 实测参数量 **64,360**（详见[§12](#12-已知事项与注意点)）。

### 5.6 `data/dataset.py` — 数据集 ✅
- `DDSPFrameDataset`：加载 `labels/*.npz` 样本，按 `sequence_length` 预建序列窗（默认 50% 重叠），
  对特征做 mean/std 标准化，不足一窗补零。
- `compute_feature_stats`：在训练集上统计逐维 mean/std。
- `collate_sequences`：堆叠为 `(B, L, ·)`。

### 5.7 `train/` — 训练 / 验证 / 导出 ✅
- `trainer.py`：`Trainer`（AdamW、梯度裁剪、可选 STFT 损失、最优/最后 checkpoint、周期渲染回调）。
  checkpoint **自包含** feature_mean/std 与 instruments，便于直接推理。
- `validate.py`：`evaluate` 返回 `harmonic_mae` / `noise_mae` / `param_loss`。
- `export.py`：导出 checkpoint、`instrument_embeddings.{npy,h}`、`feature_mean/std.{npy,h}`、`config.json`、`instruments.json`，
  以及 MCU 部署用的 **`xform_weights.bin` / `xform_weights.h`**（BNNW 网络权重，序列化顺序与 C 推理框架计算图严格一致，供 `bnn_xform_load_weights_mem` 加载）。

### 5.8 `infer.py` — 推理辅助 ✅
`load_model` / `predict_params`（标准化+clamp）/ `synth_from_params` / `render_instrument` /
`build_switch_schedule` / `streaming_render_switch`（流式乐器切换，等价固件「换嵌入下一帧生效」）。

### 5.9 `preprocess/` — 数据预处理 ✅
| 文件 | 功能 |
|---|---|
| `prepare.py` | `to_mono` · `resample`(经 audio) · `normalize_audio`(peak/rms) · `trim_silence` · `fix_length` · `prepare_wav`(一站式) |
| `align.py` | `estimate_lag`（FFT 互相关）· `shift_signal` · `align_to_reference`（源↔目标样点级对齐） |
| `segment.py` | `segment_fixed`（定长滑窗）· `segment_by_silence`（能量 VAD） |
| `qa.py` | `analyze_audio`（峰值/RMS/削波/静音/DC/NaN）· `qa_pair`（长度差/延迟/问题标记） |

## 6. 数据与缓存格式规范

**`data/raw_wav/<instrument>/<sample_id>.wav`**：48 kHz / 16-bit / mono；
同一 `sample_id` 在各乐器目录下为**同段演奏的不同音色**（音高/时序一致）。

**`manifest.csv`** 列：`sample_id, source_wav, target_wav, target_instrument`。

**`features/<sample_id>.npz`**：`features (T,20) f32`、`f0_hz (T,) f32`、`voiced (T,) f32`、`loudness_db (T,) f32`。

**`labels/<sample_id>__<target>.npz`**（训练样本单元）：
`features (T,20)`、`labels (T,40)`、`f0_hz (T,)`、`instrument_id ()` int64、`sample_id` str。

**`splits/splits.json`**：`{train:[...], val:[...], test:[...]}`（按 sample_id 划分，避免泄漏）。

**`outputs/runs/<run>/exports/`**：
- `instrument_embeddings.npy` / `.h`（`float[N][8]`）
- `feature_mean.npy` / `.h`、`feature_std.npy` / `.h`（`float[20]`）
- `xform_weights.bin` / `.h`（BNNW 网络权重；`.h` 为可 `#include` 的字节数组，供无文件系统的 MCU 直接加载）
- `config.json` / `instruments.json`
- `checkpoints/{best,last,model_final}.pt`

## 12. 已知事项与注意点

1. **`pyyaml` 未安装**：`configs/*.yaml` 的修改暂不生效（回退代码内默认值，数值一致）。
   需 `pip install pyyaml`（或改 `.json`）才能让 YAML 覆盖生效。
2. **参数量 64K vs readme 的 ~35K**：按 readme 自身列的通道数 `28→32→64→128→64→40`
   计算本就约 60K，文档中的 35K 偏乐观。64K INT8 ≈ 64 KB，仍在 <200 KB Flash 预算内。
   如需更小，可调小 `channels`（如 `[24,48,96,48]`）。
3. **控制台中文乱码**：Windows 默认 GBK，设 `PYTHONIOENCODING=utf-8` 即可正常显示（不影响功能）。
4. **`num_workers`**：Windows 下建议 0，避免多进程 DataLoader 的兼容问题。
5. **合成语料为框架验证用**：真实音质评估需接入 Ample Sound 渲染数据。
6. **torchaudio 在环境中损坏**：本框架不依赖它，无影响。

## 14. 术语表

| 术语 | 含义 |
|---|---|
| DDSP | 可微分数字信号处理；网络预测合成参数，由经典合成器生成音频 |
| FiLM | Feature-wise Linear Modulation；用条件向量逐通道缩放/平移特征 |
| 谐波幅度 | 基频整数倍处正弦分量的时域幅度（30 维） |
| 噪声子带 | 残差噪声在 Mel 间隔子带上的时域 RMS（10 维） |
| 嵌入向量 | 表征目标乐器的 8 维可学习向量；切换乐器即换它 |
| 帧/帧移 | 分析窗（1536）/ 相邻窗起点间隔（512，≈10.67ms） |
| 平行语料 | 同段 MIDI 用不同乐器渲染、音高时序对齐的成对音频 |
| OLA | Overlap-Add；重叠相加重建时域信号 |
